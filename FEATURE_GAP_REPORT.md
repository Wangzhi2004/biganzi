## 一、P0 功能（V1 必须完成）缺失清单

### 1. 作品创建器 — 部分完成

**PRD 要求：**
- 用户选择类型、输入一句话创意、目标字数、目标读者、风格
- 系统输出：作品 DNA、主角设定、核心矛盾、世界观规则、全书主线、第一卷卷纲、前 30 章规划、前 5 章场景卡、前 1 章正文草稿

**当前实现：**
- `app/project/new/page.tsx` 有创建页面
- `app/api/projects/[id]/initialize/route.ts` 有初始化 API
- `lib/ai/prompts/dna.ts` 有 DNA 生成 prompt
- `lib/orchestrator/pipeline.ts` 有完整的 9 步生成流水线

**缺失：**
- 创建页面缺少"目标字数"、"目标读者"、"风格偏好"、"是否提供样章"输入项

- 前 5 章场景卡生成未在创建流程中展示

**解决方案：**

- 在初始化流程中加入场景卡预览步骤

**当前实现：**

### 4. 正史记忆库 — 数据层完成，应用层缺失

**PRD 要求：**
- 保存人物、关系、组织、地点、道具、能力、世界规则、事件、时间线、伏笔、读者承诺、章节摘要、状态变化
- 每条正史有来源章节、可信状态、版本变更
- 每章修改后重新计算影响

**当前实现：**
- Prisma schema 定义了完整的数据模型（Character、Relationship、WorldRule、Location、Item、Ability、Organization、Event、Foreshadow、ReaderPromise、StateDiff）
- `lib/services/memory.service.ts` 有基本的 CRUD 操作

**缺失：**
- **缺少：** 正史可信状态管理（已确认/草稿/废弃/冲突的状态流转）
- **缺少：** 版本变更历史（WorldRule 有 version 字段但没有变更日志）
- **缺少：** 章节修改后自动重新计算影响范围
- **缺少：** 向量召回（PRD 提到 pgvector，当前只有 SQLite）

**解决方案：**
- 实现 `CanonStatusService` 管理正史状态流转
- 添加 `CanonVersionLog` 模型记录每次变更
- 在章节编辑保存时触发影响分析 pipeline
- P2 阶段考虑迁移到 PostgreSQL + pgvector

## 四、技术债务和已知 Bug

### 已修复
- ✅ 无限渲染循环（zustand store 订阅导致）
- ✅ useToast hook 的 useEffect 依赖问题
- ✅ chapter store 的共享 isLoading 状态问题

### 待修复
- ⚠️ 编辑器页面可能存在类似的 zustand 订阅问题（未测试）
- ⚠️ 章节树显示的章节数据结构不匹配（fetchChapterTree 返回扁平列表，组件期望树结构）
- ⚠️ 首页 `fetchProjects` 的 useEffect 依赖 `[]` 可能导致 stale closure
- ⚠️ SQLite 数据库不支持向量搜索，限制了正史记忆库的召回能力
- ⚠️ 没有用户认证系统（PRD 要求 Supabase Auth / Clerk / JWT）
- ⚠️ 没有错误边界（Error Boundary），API 错误可能导致白屏
