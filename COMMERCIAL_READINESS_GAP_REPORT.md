## 一、必须修复的问题（影响日常使用）

### 1.1 无 Error Boundary ⛔

**现状：** `.env` 中 AI API Key (`sk-4401...`) 明文写入，`.env.example` 也暴露了结构。`.gitignore` 中无 `.env` 排除规则。

**差距：**
- `.gitignore` 内容为空白行，未排除 `.env`
- AI API Key 可被任何 clone 仓库的人获取
- JWT_SECRET 为 dev 默认值

**影响：** 一旦代码上到任何公开仓库，API Key 立即泄露。

---

### 1.3 SQLite 不支持生产环境 ⛔

**现状：** 使用 `better-sqlite3` + 文件数据库 `dev.db`（1.6MB）。

**PRD 要求：** PostgreSQL + pgvector。

**差距：**
- 单文件数据库，并发写入会锁死
- 无向量搜索能力（正史记忆库召回受限）
- 无全文搜索
- 数据库迁移仅 3 个 migration，无 seed 数据
- 无法水平扩展

**影响：** 只能单用户本地开发，无法多用户部署。

---

### 1.4 无 Error Boundary ⛔

**现状：** 整个项目无 `error.tsx`、无 React Error Boundary。API 错误会导致白屏。

**影响：** 任何 AI 调用超时或后端错误直接白屏，需要手动刷新。日常使用中 AI 超时是常见场景，这是最影响体验的问题。

---

### 1.2 .gitignore 未排除敏感文件 ⚠️

**现状：** `.gitignore` 内容极简，未排除 `.env`、`dev.db`、`.next` 等。

**影响：** 如果未来推到任何仓库，API Key 会泄露。本地自用暂无风险，但建议修复。

## 五、技术债务与已知问题

### 5.1 本地自用场景下的技术选型评估

| 项目 | 现状 | 是否需要改 | 备注 |
|------|------|-----------|------|
| 数据库引擎 | SQLite | **暂不需要** | 单用户本地，SQLite 够用 |
| 向量搜索 | 无 | **可选** | 如果想做精准记忆召回可加，否则关键词匹配够用 |
| 缓存 | 无 | **暂不需要** | 单用户无并发压力 |
| 对象存储 | 无 | **暂不需要** | 本地文件系统即可 |
| 认证 | 无 | **不需要** | 本地自用 |

### 5.2 已知 Bug（影响使用）

1. **章节树数据结构不匹配** — `fetchChapterTree` 返回扁平列表，`ChapterTree` 组件期望树结构。可能导致左侧章节树显示异常
2. **首页 stale closure** — `fetchProjects` 的 `useEffect` 依赖 `[]`，新建项目后返回首页可能不刷新
3. **编辑器 zustand 订阅** — 可能存在类似已修复的无限渲染循环问题（未复现但有风险）
4. **右侧面板上下文 Tab 为空** — layout.tsx:114 中右侧面板的"章节上下文"只是占位文字
5. **伏笔状态字段大小写不一致** — 伏笔页用大写 `FULL_PAYOFF`，编辑器过滤用小写 `full_payoff`，可能导致筛选遗漏

- 架构清晰：lib/(ai/services/orchestrator) + app/api + app/project + components + stores 分层合理
- AI 网关封装好：gateway.ts 统一处理重试、超时、日志

### 问题
- **无类型安全的 API 层** — fetch 调用无类型推导，容易类型不匹配
- **状态管理碎片化** — zustand store 有 5+ 个（project/chapter/editor/generation/history），部分逻辑重复
- **无统一错误处理** — 每个页面各自 try/catch，无全局 toast 错误处理
- **硬编码样式** — 大量 inline style，CSS 变量定义不集中
- **无测试** — 0 个单元测试、0 个 E2E 测试
- **API 无版本控制** — 所有路由在 /api 下，无 /v1 前缀
- **无 API 文档** — 无 Swagger / OpenAPI

## 十、总结

本地自用场景下，当前项目已完成 PRD 中约 **70% 的 V1 功能**和 **40% 的 V2 功能**。核心链路已通：

**已能用的：**
- 作品创建 → DNA 生成 → 章节生成 → 审稿 → 确认入库 ✅
- 编辑器续写/扩写/压缩/重写/选中改写 ✅
- 风格指纹 + 自动偏移检测 ✅
- 人物/伏笔/世界观 CRUD ✅
- 套路变体/钩子诊断/读者模拟/关系图/时间线/规划器 ✅

**影响日常使用的缺口：**
1. **Error Boundary 缺失** — AI 超时白屏，最影响体验
2. **伏笔操作粗糙** — 只能改状态，无加深/误导/回收方案等专用操作
3. **编辑器缺两个操作** — "改文风"和"改情节"
4. **人物功能不完整** — 无详情页、无弧线、无出场列表
5. **已知 Bug** — 章节树结构不匹配、右侧面板空内容

**建议路线：** Phase 1（3-5天）解决稳定性问题，Phase 2（1-2周）补全核心体验，Phase 3（1周）打磨细节。总计约 2-3 周可达"日常好用"状态。SQLite 本地够用，无需迁移数据库。
